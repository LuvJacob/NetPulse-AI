Product Requirements Document (PRD)
AI-Powered Network Monitor Dashboard
1. Overview
Product Name

NetPulse AI
(working title)

Product Summary

NetPulse AI is a lightweight network monitoring dashboard that continuously checks the availability and latency of websites, servers, and devices. It logs uptime metrics, detects outages, visualizes network health, and uses AI to summarize performance issues and anomalies.

The project is designed as a portfolio-ready systems/network engineering application that demonstrates:

Python backend development
Networking fundamentals
Monitoring systems
Automation
Data persistence
Dashboard development
AI integration
2. Goals
Primary Goals
Monitor multiple network targets automatically
Store and visualize monitoring data
Detect outages and high latency
Provide AI-generated network health summaries
Secondary Goals
Build a résumé-worthy infrastructure project
Learn Flask and backend architecture
Practice systems engineering concepts
Gain experience with APIs and AI integration

Students or junior IT professionals wanting to monitor:

websites

5. Functional Requirements
AI Integration

The system should:

Send recent monitoring logs to Gemini
Generate readable network health summaries
Analyze trends in latency and outages
Return concise explanations suitable for dashboard display

The system must:

Securely store the Gemini API key in a .env file
Handle API failures gracefully
Keep AI logic separated inside ai_service.py
